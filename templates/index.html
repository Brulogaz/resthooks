The real-time web is here but REST APIs haven't kept up.

Many major players have already begun standardizing on subscription webhooks.
REST hooks consolidate this momentum and push it to a broader audience.

  • 1. REST hooks are 66x more efficient than traditional polling.

    • Case study: Zapier is a SaaS service which connects hundreds of other web apps together online. Polling is a necessity because it is the most ubiquitous way to detect changes.

    Over a representative time period, Zapier polled for changes 30 million times but only took action on 460,000 of those polls (1.5% efficient).

    Contrast this with REST hooks, where 547,000 hooks were received over the same period and all of them were acted upon (100% efficient).

    In other words, if everyone implemented REST hooks, server load for both the sender and receiver could be reduced by 66x.

  • 2. User conversion rates improved 24% with REST hooks.

    • Case study: Wufoo is a popular survey product service which integrates with Zapier. Wufoo supports both classic webhooks (the user must copy and paste a URL from one app to the other) and REST hooks (the webhook is automatically created).

    Users on Zapier tried to set up an integrations with Wufoo using classic webhooks and were successful 95 times out of 215 tries, a 44% conversation rate.

    When users tried REST hooks, however, 261 of 477 were successful, a 54.7% conversion. REST hooks improved conversion rates by 24%.

    With REST hooks, user experience is also greatly improved because users don't need to go offsite to set up an integration.

  • 3. When asked, over 80% of developers prefer hooks.

    • Survey: When a group of 150 developers were surveyed, 80% preferred to consume a 3rd party REST API by being notified of changes via webhooks instead of polling.

    Polling requires developers to maintain state and infrastructure. There is such unanimous support for real-time change notifications because polling is very difficult and time intensive.

    Developers can spend their time doing meaningful things with your REST API instead of fighting the quirks of polling.

REST hooks reduce server load, make devs happy, and improve conversion.
